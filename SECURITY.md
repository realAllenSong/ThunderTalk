# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in ThunderTalk, please report it responsibly.

**Do not open a public GitHub issue for security vulnerabilities.**

Instead, please email: [security contact TBD]

We will acknowledge receipt within 48 hours and provide a timeline for a fix.

## Scope

ThunderTalk processes audio locally on-device. Security concerns include:

- Local file access and permissions
- Clipboard content handling
- Model download integrity (SHA256 verification)
- System-level keyboard/audio access
